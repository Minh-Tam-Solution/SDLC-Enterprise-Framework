# AGENTS.md Template

**Version**: 1.0.0
**Status**: RECOMMENDED (replaces MTS/BRS/LPS)
**Framework**: SDLC 5.1.3
**ADR Reference**: ADR-029

---

## Overview

`AGENTS.md` is the **industry standard** for providing context to AI coding assistants (Cursor, Claude Code, GitHub Copilot). It is committed to the repository root and read by AI tools to understand project conventions.

**Key Benefits**:
- Industry standard (60K+ projects on GitHub)
- Vendor-neutral (works with any AI tool)
- Simple (single file, ≤150 lines recommended)
- Auditable (committed to git)

---

## AGENTS.md replaces:

| Old Artifact | AGENTS.md Section | Notes |
|--------------|-------------------|-------|
| **MTS (MentorScript)** | `## Conventions`, `## DO NOT` | Static standards |
| **BRS (BriefingScript)** | `## Quick Start`, `## Architecture` | Project overview |
| **LPS (LearnedPatternsScript)** | _Context Overlay Service_ | Delivered via PR comments (NOT committed) |

## Template

```markdown
# AGENTS.md - [Project Name]

This file provides context for AI coding assistants (Cursor, Claude Code, Copilot).
Keep ≤150 lines. Dynamic context is delivered via PR comments.

## Quick Start

- `docker compose up -d` - Start all services
- `npm run dev` - Start frontend dev server
- See README.md for full setup

## Architecture

- **Backend**: Python 3.11+ / FastAPI
- **Frontend**: React 18 / TypeScript
- **Database**: PostgreSQL 15
- **Cache**: Redis 7
- **Services**: [docker-compose.yml services list]

## Current Stage

_Note: Current sprint/stage context is delivered dynamically via PR comments._
_Check `.sdlc-config.json` for SDLC tier and stage mapping._

## Conventions

**Python:**
- snake_case for files and functions
- Type hints required (Python 3.11+)
- Docstrings: Google style
- Linting: ruff (see ruff.toml)

**TypeScript:**
- PascalCase for React components
- camelCase for utilities
- Strict mode enabled (tsconfig.json)

**Git:**
- Branch naming: `feature/`, `fix/`, `chore/`
- Commit format: `type(scope): description`
- PR required for all changes

## Security

- **NEVER** commit secrets (API keys, passwords)
- Use environment variables for configuration
- AGPL Containment: Network-only access to AGPL components
  - MinIO: Use S3 API via HTTP (no SDK import)
  - Grafana: Embed via iframe only
- Follow OWASP Top 10 guidelines

## DO NOT

- Add TODO comments or placeholder code (Zero Mock Policy)
- Skip error handling
- Hardcode secrets or environment-specific values
- Import AGPL libraries directly (use network APIs)
- Commit without running tests
- Push directly to main branch

---

_Generated by sdlcctl agents init | YYYY-MM-DD_
```

---

## Guidelines

### Line Limit
- **Recommended**: ≤150 lines
- **Maximum**: 200 lines
- **Reason**: AI context windows are limited; too much context = ignored

### Sections to Include

| Section | Required | Purpose |
|---------|----------|---------|
| Quick Start | ✅ | How to run the project |
| Architecture | ✅ | System overview |
| Conventions | ✅ | Code standards |
| Security | ✅ | Security rules |
| DO NOT | ✅ | Anti-patterns to avoid |
| Current Stage | Optional | Static note about dynamic delivery |
| Git Workflow | Optional | Branch/commit conventions |

### What NOT to Include

- ❌ **Secrets** (API keys, passwords, tokens)
- ❌ **Sprint-specific context** (use Context Overlay instead)
- ❌ **Full documentation** (link to /docs instead)
- ❌ **Generated code** (examples only)

## Validation

### CLI Validation

```bash
# Generate AGENTS.md
sdlcctl agents init

# Validate existing AGENTS.md
sdlcctl agents validate AGENTS.md

# Lint and auto-fix
sdlcctl agents lint AGENTS.md --fix
```

### Pre-commit Hook

Add to `.pre-commit-config.yaml`:

```yaml
repos:
  - repo: local
    hooks:
      - id: agents-md-validate
        name: Validate AGENTS.md
        entry: sdlcctl agents validate AGENTS.md
        language: system
        pass_filenames: false
        files: AGENTS.md
```

## Dynamic Context (Context Overlay)

For **runtime context** (current sprint, gate status, active constraints), use the **Context Overlay Service**:

```bash
# Get context via CLI
sdlcctl agents context --project-id <uuid>

# Get context for PR comment
curl "https://api.sdlc.example.com/v1/context/<project_id>?format=pr_comment"
```

**Context Overlay is NOT committed to git.** It's delivered via:
- PR comments (auto-posted on PR open)
- CLI output
- GitHub Check Run output
- VS Code Extension panel

## Migration from MTS/BRS/LPS

### Step 1: Generate AGENTS.md

```bash
cd your-project
sdlcctl agents init --force
```

### Step 2: Review and customize

Edit the generated AGENTS.md to include project-specific conventions.

### Step 3: Archive old artifacts

```bash
mkdir -p docs/99-Legacy/SASE
mv MentorScript.md docs/99-Legacy/SASE/
mv BriefingScript.md docs/99-Legacy/SASE/
mv LearnedPatternsScript.md docs/99-Legacy/SASE/
```

### Step 4: Commit

```bash
git add AGENTS.md
git commit -m "feat: Add AGENTS.md (replaces MTS/BRS/LPS per ADR-029)"
```
